Ways we can implement accessibility

There are many simple ways in which designers can design their websites to be accessible, this page will be all about showing some of these design techniques. This website itself is designed using many of the techniques discussed.

Don Normans Principles of design

Don Normans principles of design are not specific techniques, but they serve as a general rule of thumb to follow whenever designing anything. Adhering to these principles will always help designes be more accessible. Normans principles are the following:

  • 1. Visibility
  • 2. Feedback
  • 3. Constraints
  • 4. Mapping
  • 5. Consistency
  • 6. Affordance

HTML5 semantic structure

One of the most important features of HTML5 is its semantics. Semantic HTML refers to syntax that makes the HTML more comprehensible by better defining the different sections and layout of web pages. It makes web pages more informative and adaptable, allowing browsers and search engines to better interpret content. These are some of the semantic elements that can be used to define things on your website:

  • <article>
  • <aside>
  • <details>
  • <figcaption>
  • <figure>
  • <footer>
  • <header>
  • <main>
  • <mark>
  • <table>
  • <form>

Color contrast

Making sure that your website has good color contrast is essential in order for the website to both look appealing, but also be accesible. Also, ensuring that the colors you are using are color blind safe is very important too. A great tool for this is the adobe colors website.

For example, I used the "Accessibility tools" section of the website to check the color of the visited links in the nav bar, with the background color.

Screen reader only

Many people rely on screenreaders to interact with the content on webpages. Using css, we can hide elements visually but not for screen readers, ensuring that people using screen readers have access to this content.

Skip navigation bar

Another element we can implement using css is creating a pop up when tabbing through a website which gives the option for users to skip the navigation bar and go straight to the main content.

If you press tab right now, you will see exactly what it looks like!

How do you make sure its accessible?

What if you have desgined your website, and you think you did a good job in making it accessible but arent completely sure? Like adobe colors, there are many online resources you can use to validate your website, and ensure lives up to your expectations.

Here are three incedbile resources for checking your webpage:

  • 1. HTML validator
  • 2. HTML outliner
  • 3. WAVE

HTML Validator

If you visit this link, it will take you to the Nu html checker. On this site, you can upload a file, or even copy and paste all of your code into the text box. Once you do this, the program will run and you will receive feedback on your website. It will check your html, and tell you if it found any errors in your code. HTML Outliner

The outliner is a chrome extension that shows the user the structure of their website. This helps with finding any untitled elements. The extension can be found in this link.

WAVE

Wave is another browser extension, like the html outliner. It provides visual feedback about the accessibility of your web content by injecting icons and indicators into your page. No automated tool can tell you if your page is accessible, but WAVE facilitates human evaluation and educates about accessibility issues. All analysis is done entirely within the Chrome browser allowing secure valuation of intranet, local, password protected, and other sensitive web pages.
